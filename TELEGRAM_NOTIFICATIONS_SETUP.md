# 📱 Настройка Telegram уведомлений

## ✅ Что уже реализовано

Функциональность Telegram уведомлений **уже полностью реализована** в проекте:

1. ✅ **Уведомление о новой регистрации** - отправляется автоматически при регистрации нового пользователя
2. ✅ **Уведомление об успешной оплате** - отправляется автоматически при успешной оплате подписки

## 🔧 Что нужно настроить

Для работы уведомлений нужно настроить Telegram бота и добавить переменные окружения.

### Шаг 1: Создайте Telegram бота

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям:
   - Придумайте имя для бота (например: "NinjaTraining Notifications")
   - Придумайте username для бота (должен заканчиваться на `bot`, например: `ninja_training_notifications_bot`)
4. **Скопируйте токен бота** - он будет выглядеть примерно так: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`

### Шаг 2: Получите Chat ID

Есть несколько способов получить Chat ID:

#### Способ 1: Личный чат с ботом
1. Найдите вашего бота в Telegram (по username, который вы создали)
2. Начните диалог с ботом, отправив любое сообщение (например: `/start`)
3. Откройте в браузере: `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
   - Замените `<ВАШ_ТОКЕН>` на токен, который вы получили от BotFather
4. Найдите в ответе поле `"chat":{"id":123456789}` - это и есть ваш Chat ID

#### Способ 2: Использовать бота @userinfobot
1. Найдите бота [@userinfobot](https://t.me/userinfobot) в Telegram
2. Начните диалог с ним
3. Он покажет ваш Chat ID

#### Способ 3: Для группы/канала
1. Добавьте бота в группу или канал
2. Сделайте бота администратором (если это канал)
3. Отправьте сообщение в группу/канал
4. Откройте: `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
5. Найдите `"chat":{"id":-123456789}` - отрицательное число для группы/канала

### Шаг 3: Добавьте переменные в .env файл

Откройте файл `.env` в корне проекта и добавьте следующие строки:

```env
# Настройки Telegram для уведомлений
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
TELEGRAM_CHAT_ID=123456789

# HTTP-прокси для Telegram (если прямой доступ недоступен)
TELEGRAM_PROXY_HOST=170.83.233.92
TELEGRAM_PROXY_PORT=8000
TELEGRAM_PROXY_USERNAME=DcNVAT
TELEGRAM_PROXY_PASSWORD=32DeAo
```

**Где:**
- `TELEGRAM_BOT_TOKEN` - токен бота от BotFather
- `TELEGRAM_CHAT_ID` - ID чата, куда отправлять уведомления (ваш личный ID, ID группы или канала)
- `TELEGRAM_PROXY_*` - настройки HTTP-прокси для запросов к Telegram API

### Webhook для статистики: быстрый ответ и таймауты nginx

Эндпоинт `POST /telegram/webhook` **сразу** возвращает Telegram `{"ok": true}`, а запросы к БД и отправка ответа в чат идут **в фоне**. Так реже ловятся обрывы из‑за долгого запроса и повторные доставки одного и того же update.

Если перед приложением стоит **nginx**, имеет смысл увеличить таймауты для префикса `/telegram/`:

```nginx
location /telegram/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_read_timeout 120s;
    proxy_connect_timeout 10s;
    proxy_send_timeout 120s;
}
```

### Режим long polling (как «настоящий» бот без входящего webhook)

Если **входящие** запросы от Telegram до сервера нестабильны, можно не использовать webhook для получения сообщений: процесс сам опрашивает Telegram методом `getUpdates` (исходящий HTTPS, часто это проще пробить с сервера).

В `.env`:

```env
TELEGRAM_UPDATES_MODE=polling
```

После перезапуска в логах появится строка про режим long polling. При старте вызывается `deleteWebhook`, то есть webhook у бота снимается.

Важно: не включайте одновременно осмысленный webhook в BotFather и несколько инстансов с `polling` без понимания — возможны пропуски или дубли.

### Шаг 4: Перезапустите сервер

После добавления переменных окружения перезапустите сервер:

```bash
# Остановите сервер (Ctrl+C)
# Затем запустите снова
uvicorn app.main:app --reload
```

При регистрации нового пользователя вы получите сообщение:
```

📧 Email: user@example.com
👤 Логин: username
```

При успешной оплате подписки вы получите сообщение:

## 🔍 Проверка работы

1. **Проверьте логи при запуске сервера:**
   - Если видите: `TelegramService инициализирован успешно` - всё настроено правильно
   - Если видите: `Telegram уведомления отключены: не настроены TELEGRAM_BOT_TOKEN или TELEGRAM_CHAT_ID` - проверьте переменные окружения

2. **Протестируйте регистрацию:**
   - Зарегистрируйте тестового пользователя через API
   - Проверьте, пришло ли уведомление в Telegram

3. **Протестируйте оплату:**
   - Создайте тестовый платеж
   - После успешной оплаты проверьте уведомление в Telegram

## ⚠️ Важные замечания

- Если переменные `TELEGRAM_BOT_TOKEN` или `TELEGRAM_CHAT_ID` не настроены, уведомления просто не будут отправляться, но это не повлияет на работу приложения
- Ошибки отправки уведомлений логируются, но не прерывают основной процесс (регистрацию или обработку платежа)
- Для продакшена рекомендуется использовать отдельный канал для уведомлений, а не личный чат

## 🐛 Решение проблем

### Уведомления не приходят
1. Проверьте, что токен бота правильный (без лишних пробелов)
2. Проверьте, что Chat ID правильный (число без кавычек)
3. Убедитесь, что вы начали диалог с ботом (если используете личный чат)
4. Проверьте логи сервера на наличие ошибок

### Ошибка "Unauthorized"
- Проверьте правильность токена бота
- Убедитесь, что бот активен

### Ошибка "Chat not found"
- Проверьте правильность Chat ID
- Убедитесь, что бот добавлен в группу/канал (если используете группу/канал)
- Убедитесь, что вы начали диалог с ботом (если используете личный чат)
